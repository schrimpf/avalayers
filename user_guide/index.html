avalayers User Guide

This guide provides a comprehensive walkthrough for using the avalayers package to model avalanche flow dynamics with integrated forest friction.

1. Workflow Overview

The avalayers pipeline follows a 5-step process: 1. Download: Acquire DTM (Terrain) and DSM (Surface) tiles. 2. Prepare: Merge tiles, calculate Canopy Height (CHM), and define Release Areas. 3. Simulate: Run the FlowPy engine with custom friction parameters. 4. Visualize: Inspect results in an interactive browser-based dashboard. 5. Export: Generate .kmz file Google Earth and other services.

2. Step-by-Step Tutorial

1. Acquire Elevation Data (DTM & DSM)

Start by downloading the bare-earth DTM (FABDEM) and surface DSM (Copernicus) for your region.

uv run -m avalayers download
or with a pre-defined bounding box: 
uv run -m avalayers download --bbox -123.1488 49.6315 -123.0692 49.6684
Note: This creates a data/dems/ directory containing the merged .tif files. The FABDEM data files cover a large area and take some time to download. They will be saved and re-used.

💡 Pro Tip: Custom Input Data & Resolution

While the download command provides convenient global coverage via FABDEM and Copernicus GLO-30 (~30m), simulation accuracy is significantly enhanced by using higher resolution data (e.g., 1m to 5m LiDAR).

If you have access to local high-resolution DTM/DSM tiles (e.g., from OpenTopography or government agencies): 1. Skip the download command. 2. Provide your own .tif files directly to the prepare command. 3. Optional: to reduce computation time, use the --res flag during prepare to downsample to a lower resolution (5m or 10m seem to work well).

[!IMPORTANT] Ensure your custom rasters are in a projected coordinate system (like UTM) for accurate slope and friction calculations.

Step 2: Preparing the Project

The prepare command sets up the directory structure that FlowPy requires. Note: you will have to change the dtm and dsm filenames to match what you downloaded.

uv run -m avalayers prepare \
    --dtm data/dems/fabdem_dtm_-123.1488_49.6315_-123.0692_49.6684.tif \
    --dsm data/dems/copernicus_glo30_dsm_-123.1488_49.6315_-123.0692_49.6684.tif \
    --out MyProject

Interactive Release Zone Tuning

During preparation, avalayers calculates the Slope and Elevation of your terrain. This is used to determine the release zone. The release zone will consists of all cells whose elevation is above a specified altitude, whose slope is in a range, and whose forest coverage is below a specified threshold.
A map will open in your browser showing the release zone. You can choose to modify the release zone parameters directly from the terminal until you are satisfied.

Forest Density (FSI)

The Forest Structure Information (FSI) layer is automatically generated by taking DSM - DTM (Canopy Height) and normalizing it. - You can control the normalization threshold with --tree-height-limit (default 30m). - Trees taller than this limit result in an FSI of 1.0 (fully dense).

Step 3: Running the Simulation

The simulate command executes the FlowPy model on your prepared project.

uv run -m avalayers simulate --project MyProject --max-forest-friction 20.0 --overwrite

Key Parameters: - --max-forest-friction: The additional friction angle (alpha) added when FSI = 1.0. Increase this if the flow passes through trees too easily. - --forest-vel-thresh: The velocity (m/s) below which forest friction is most effective. - --overwrite: Forcefully re-run a simulation even if outputs already exist.

Step 4: Visualization

Generate a comprehensive dashboard of all simulation results.

uv run -m avalayers visualize --project MyProject --opacity 0.5 --browser

The dashboard includes: - DTM: The underlying terrain. - REL: The release zones you defined. - FSI: The forest coverage used for friction. - zdelta: peak flow thickness (Max Depth). - Interactive Toggles: Turn layers on/off to see how the flow interacts with trees.

Step 5: Google Earth Export

For a 3D perspective, export the layers to KMZ.

uv run -m avalayers export --project MyProject --open

3. Detailed CLI Reference

avalayers prepare

Option Default Description
--res native Force downsampling to a specific resolution (e.g. 5.0m).
--min-slope 30.0 Minimum slope angle (deg).
--max-tree-cov 0.1 Max FSI allowed in a release cell.
--tree-height-limit 30.0 Max height (m) for FSI normalization.

avalayers simulate

Option Default Description
--max-forest-friction 10.0 Added friction in degrees for dense canopy.
--forest-vel-thresh 30.0 Critical velocity for friction damping.

4. Scientific Background: Forest Friction

avalayers uses the ForestFriction module from AvaFrame. See the AvaFrame documentation. This module adds a resistive term to the shallow water equations governing the avalanche flow. The resistance is scaled by the FSI value of the cell:

\[ \text{Friction}_{total} = \text{Coulomb}_{friction} + \text{FSI} \times \text{ForestFriction}_{tuning} \]

Dense forests (FSI \(\approx\) 1) significantly increase the "alpha-angle" of the flow, effectively slowing it down or stopping it in sheltered terrain.
